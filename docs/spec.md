# Camp Note 設計仕様書（ドラフト）

## 1. プロダクト概要
What：キャンプ準備〜記録〜ふりかえりを1つのHTMLアプリで管理
Why：忘れ物・失敗の再発防止、記録の一元化
For Whom：開発者本人と身近なキャンパー

## 2. UX設計原則
- オフライン前提（保存はブラウザ内。ネット不要で使える。AI機能のみオンライン必須）
- ワンページ・タブ切替型（8タブ、下部ナビ）
- 時間帯に応じた背景テーマ（朝/昼/夕/夜）で世界観演出

## 3. 機能要件と画面一覧
| タブ | 機能 |
|---|---|
| 持ち物(pack) | カテゴリ別持ち物リスト、チェック管理 |
| 買い物(shop) | 買い物リスト管理 |
| 在庫(stock) | 名前・数量・状態・期限を持つ在庫管理 |
| 失敗ログ(fail) | カテゴリ・タイトル・内容・対策を記録 |
| キャンプログ(log) | 日付・名前・メモ・評価・写真を記録 |
| ふりかえり(review) | 集計統計（訪問回数・平均評価・お気に入り・失敗傾向） |
| 口コミ(kuchi) | キャンプ場ごとの口コミ投稿・AI要約・AI名寄せ |
| 行きたい場所(wish) | ウィッシュリスト |

## 5. データモデル
DBなし。ブラウザのlocalStorageのみで完結：
- `cn4`：本体状態（持ち物・買い物・在庫・失敗・ログ・写真等）
- `cn4_k`：口コミ配列

※端末ローカルのみで、他端末・他人とはデータが共有されない点に注意。
**【要判断】一般公開を目指す場合、この設計のままだと「機種変更・ブラウザのデータ削除で記録が全部消える」問題が起きる。将来クラウド保存（例：Supabase）へ移行するかはRiriko本人の判断待ち（Q&Aで確認中）**
